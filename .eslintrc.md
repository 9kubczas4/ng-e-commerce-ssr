# ESLint Configuration

This project uses ESLint with the `eslint-plugin-boundaries` to enforce architectural boundaries.

## Architectural Boundaries

The following import rules are enforced:

### Core (`src/app/core/**`)
- ✅ Can import from: `core`, `shared`
- ❌ Cannot import from: `features`

### Shared (`src/app/shared/**`)
- ✅ Can import from: `shared`
- ❌ Cannot import from: `core`, `features`

### Features (`src/app/features/**`)
- ✅ Can import from: `core`, `shared`, same feature
- ❌ Cannot import from: other features

## Example Violations

```typescript
// ❌ BAD: Feature importing from another feature
// src/app/features/products/product.service.ts
import { BasketService } from '../basket/basket.service';

// ✅ GOOD: Feature importing from core
// src/app/features/products/product.service.ts
import { BasketService } from '@app/core/services/basket.service';

// ❌ BAD: Shared importing from core
// src/app/shared/components/button.component.ts
import { ThemeService } from '@app/core/services/theme.service';

// ✅ GOOD: Core importing from shared
// src/app/core/layout/header.component.ts
import { ButtonComponent } from '@app/shared/components/button.component';
```

## Running Linting

```bash
# Check for linting errors
npm run lint

# Fix auto-fixable linting errors
npm run lint:fix

# Check code formatting
npm run format:check

# Fix code formatting
npm run format
```

## Accessibility Rules

The ESLint configuration includes accessibility rules from `@angular-eslint/template` to ensure WCAG AA compliance:

- Alt text for images
- Label associations for form controls
- Valid ARIA attributes
- Keyboard event handlers for mouse events
- And more...

## TypeScript Rules

- Strict type checking enabled
- No explicit `any` types allowed
- Unused variables must be prefixed with `_`
- Angular component/directive selectors enforced
